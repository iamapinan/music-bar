# แผนงานการปรับปรุง Seek Bar ตัวเล่นเพลงให้สามารถลากเวลาได้และมีพื้นที่สัมผัสระดับพรีเมียม

แผนงานนี้มีเป้าหมายเพื่อแก้ไขปัญหาแถบความคืบหน้า (Seek Bar) ไม่สามารถกดหรือลากเวลาเล่นเพลงได้ (โดยเฉพาะบนอุปกรณ์พกพาและบางหน้าจอ) เนื่องจากพื้นที่สัมผัสมีความสูงน้อยเกินไป (4px) และมีการใช้คลาสซ้อนทับกันที่บดบัง Pointer Events

---

## 🛠️ รายการเปลี่ยนแปลงที่เสนอ (Proposed Changes)

### 1. ปรับปรุงการจัดวางเลย์เอาต์และการทำงานของ Seek Bar

#### 📂 [MODIFY] [player-bottom-bar.tsx](file:///Users/apinan/Developments/music-bar/components/player-bottom-bar.tsx)
- ลบการเรนเดอร์โครงสร้างที่ซ้อนทับกัน (ลบ Custom Div แสดงผลสีแถบที่มี `bg-primary/10` ออก) เพื่อใช้ความสามารถดั้งเดิมของ Radix Slider ในการเรนเดอร์แถบและสีที่แม่นยำกว่า 100%
- เพิ่มพื้นที่สัมผัสของแถบ Seek Bar จากความสูง `h-1` (4px) ขึ้นเป็น `h-5` (20px) เพื่อความสะดวกสบายในการใช้นิ้วแตะหรือใช้เมาส์ลากบนทุกอุปกรณ์
- ใช้ประโยชน์จากคลาสลูกของ Slider ใน Tailwind CSS เพื่อจัดแต่งดีไซน์มินิมัลลิสต์หรูหราสไตล์ Spotify:
  - ขนาดความหนาของเส้นปกติคือ `h-1` และขยายเป็น `h-1.5` เมื่อ Hover (`group-hover:[&_[data-slot=slider-track]]:h-1.5`)
  - ซ่อนปุ่มจับลากทรงกลม (Thumb) เป็น `opacity-0` เป็นค่าเริ่มต้น และจะแสดงขึ้นมาเป็น `opacity-100` สวยงามเฉพาะเมื่อมีการสัมผัสหรือ Hover ที่ Seek Bar เท่านั้น
  - การันตีความเข้ากันได้ของ Pointer Events ด้วยการขจัดคลาสที่ซ้อนทับและใช้คุณสมบัติของ Radix UI โดยตรง
