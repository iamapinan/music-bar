# แผนการเพิ่มปุ่มบังคับอัปเดตแอปพลิเคชัน (Force App Update Plan)

แผนงานนี้จัดทำขึ้นเพื่อเพิ่มปุ่มและฟังก์ชันในการ "บังคับอัปเดตแอป" (Force Update) เพื่อให้ผู้ดูแลระบบ ลูกค้า และเครื่องเล่นเพลงหลักสามารถตรวจสอบและบังคับดึงข้อมูลแอปพลิเคชันเวอร์ชันใหม่จากเซิร์ฟเวอร์ และล้างแคชเก่าที่ตกค้างในเบราว์เซอร์ได้ทันทีโดยไม่ต้องรอระบบตรวจสอบอัตโนมัติ

---

## หัวข้อที่ต้องการการรีวิวจากผู้ใช้งาน (User Review Required)

> [!IMPORTANT]
> - **การล้างแคชและบังคับอัปเดต (Force Reload & Cache Clearing):**
>   - ฟังก์ชันบังคับอัปเดตจะสั่งงาน Service Worker ให้ตรวจสอบเวอร์ชันใหม่ หากตรวจพบ จะเรียก `SKIP_WAITING` และทำการรีเฟรชหน้าเว็บทันที
>   - ในกรณีที่ไม่พบเวอร์ชันใหม่จาก Service Worker ระบบจะทำการล้าง Cache Storage ทั้งหมดที่สร้างขึ้นโดยเบราว์เซอร์ และบังคับโหลดหน้าเว็บใหม่จากเซิร์ฟเวอร์โดยตรง เพื่อป้องกันอาการไฟล์เว็บเก่าค้าง (Stale Cache) และสร้างความมั่นใจว่าผู้ใช้จะได้เวอร์ชันล่าสุด 100%
> - **การสื่อสารผ่าน UI:**
>   - ทุกกระบวนการจะแสดงสถานะผ่านกล่องข้อความแจ้งเตือน (Toast) ของระบบ เช่น "กำลังตรวจสอบเวอร์ชันใหม่..." "กำลังอัปเดตเป็นเวอร์ชันใหม่..." หรือ "แอปพลิเคชันเป็นเวอร์ชันล่าสุดแล้ว" เพื่อให้ผู้ใช้ทราบสถานะอย่างต่อเนื่องโดยไม่มีการใช้ Emoji ใน UI

## รายการไฟล์ที่จะสร้างและแก้ไข (Proposed Changes)

### 1. ฟังก์ชันช่วยเหลือสำหรับการอัปเดตแอป (Utility)

#### [NEW] [app-update.ts](file:///Users/apinan/Developments/music-bar/lib/app-update.ts)
- สร้างฟังก์ชัน `forceUpdateApp(setIsUpdating?: (loading: boolean) => void)`
- ดึงทะเบียนของ Service Worker ปัจจุบันมาเช็คและสั่ง `.update()`
- หากพบการอัปเดตที่รออยู่ (`waiting`) ให้ทำการส่งข้อความ `SKIP_WAITING` และสั่ง reload
- หากไม่พบการอัปเดต ให้ล้าง Cache Storage ทั้งหมดด้วย `caches.delete` และทำ hard reload เพื่ออัปเดตไฟล์ assets
