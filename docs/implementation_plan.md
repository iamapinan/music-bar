# แผนการปรับปรุงธีมสว่างสำหรับส่วนควบคุมผู้ดูแลระบบแบบสมบูรณ์ 100% (Complete Light Theme Enforcement for Admin)

แผนงานนี้จัดทำขึ้นเพื่อการันตีว่า หน้าผู้ดูแลระบบ (`/admin`) จะเป็นธีมสว่างอย่างสมบูรณ์แบบ 100% โดยจะจัดการกับองค์ประกอบภายนอกโครงสร้างหลัก เช่น Radix Portals (กล่องหน้าต่าง Modal, เมนู Select Dropdown), แถบเลื่อนหน้าจอ (Scrollbars), และกล่องแจ้งเตือน (Sonner Toasts) ที่ปกติอาจจะได้รับผลกระทบจากการตั้งค่าคลาส `dark` ทั่วระบบในระดับ `html`

---

## หัวข้อที่ต้องการการรีวิวจากผู้ใช้งาน (User Review Required)

> [!IMPORTANT]
> - **การถอดและสลับคลาส `dark` อัตโนมัติ:** เมื่อผู้ใช้เปิดเข้าหน้า `/admin` ระบบจะทำการถอดคลาส `dark` ออกจากอิลิเมนต์ `<html>` ทันทีผ่าน `useEffect` และจะคืนค่าใส่คลาส `dark` กลับคืนเมื่อนำออกจากหน้า เพื่อรักษาประสบการณ์ดนตรีในธีมมืด (Dark Ambient) ของหน้าหลักฝั่งลูกค้าตามปกติ
> - **การเพิ่มคลาส `admin-mode` บน `html` และ `body`:** เพื่อป้อนค่าตัวแปร CSS ธีมสว่างให้กับหน้าต่าง Radix Portals และกล่องแจ้งเตือนภายนอกโครงสร้าง ให้ได้รับผลกระทบเป็นธีมสว่างสอดคล้องกันทั่วถึง
> - **ปรับดีไซน์ Toasts และ Scrollbars:** ปรับเปลี่ยนกล่อง Sonner Toasts และแถบเลื่อน Scrollbars ภายในระบบแอดมินให้เป็นโทนสีขาว/เทาสว่างพรีเมียม

- กำหนดสไตล์แถบเลื่อนหน้าจอ `body.admin-mode ::-webkit-scrollbar-thumb` เป็นโทนสีเทาสว่างกลมกลืน

3. ทดลองเพิ่มเพลงหรือดำเนินรายการ และดูว่ากล่องแจ้งเตือน Sonner Toasts ที่แสดงขึ้นมาเป็นสีขาวสว่างตามธีม
